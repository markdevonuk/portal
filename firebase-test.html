Firebase Security Rules Helper

1. Diagnose Registration Issue

If you're getting a "database access denied" error during registration, it's likely due to Firebase security rules preventing access to the allowedEmails collection.

Common issue: The security rules may be restricting read access to the allowedEmails collection for unauthenticated users.

Recommended Firestore Rules

// Allow read access to allowedEmails collection for everyone service cloud.firestore { match /databases/{database}/documents { // Allow access to allowedEmails collection for checking during registration match /allowedEmails/{email} { allow read: if true; // Allow public read access allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.isAdmin == true; } // User profiles - users can read/write their own data match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; allow read, write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.isAdmin == true; } // Profiles collection match /profiles/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; allow read, write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.isAdmin == true; } // Default deny all other access match /{document=**} { allow read, write: if false; } } }

The key is allowing public read access to the allowedEmails collection.

4. How to Update Firestore Rules

To update your Firebase security rules:

  1. Go to the Firebase Console
  2. Select your project: fmsportal-dbfb3
  3. Navigate to Firestore Database → Rules
  4. Replace the current rules with the recommended rules above
  5. Click "Publish"
Note: Changes to security rules can take a few minutes to propagate.
